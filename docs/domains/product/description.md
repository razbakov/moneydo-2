# Product — Domain Description (Delegation Canvas)

**Date/Version:** 2026-04-05
**Delegator:** Alex Razbakov (Founder)
**Review date:** 2026-07-05

## Purpose
**Primary Driver:** Users need a clear, mindful way to plan budgets and track expenses, but the product's feature set, user experience, and roadmap need intentional curation to stay simple and Kakeibo-aligned.
**Main Requirement:** Define what gets built, ensure features serve the organizational driver, and maintain product-market fit through user feedback and experimentation.

## Key Responsibilities
- Maintain and prioritize the product backlog
- Write user stories with acceptance criteria (BDD scenarios)
- Define the user journey and story map
- Conduct lightweight user research (feedback analysis, usage patterns)
- Ensure every feature traces back to the primary driver
- Say "no" to feature requests that add complexity without serving the Kakeibo philosophy

## Customers and Deliverables
| Customer | Deliverable |
|----------|-------------|
| Engineering domain | Prioritized backlog with BDD scenarios |
| Alex (Founder) | Product recommendations with rationale |
| End users (indirectly) | Coherent, simple product experience |

## Dependencies
| Provider | What they deliver |
|----------|-------------------|
| Alex (Founder) | Strategic direction, approval on scope changes |
| Growth domain | User feedback, search intent data |
| Engineering domain | Feasibility assessments, technical constraints |

## External Constraints
- Must stay aligned with Kakeibo methodology (four categories, mindfulness, reflection)
- No features that compromise user privacy
- Subscription price fixed at 3€/month — features must justify this, not more

## Key Challenges
- Resisting feature creep while keeping the product competitive
- Understanding user needs without a large user base for research
- Balancing simplicity with enough functionality to retain subscribers

## Key Resources
- Firebase Analytics for usage data
- User feedback (support emails, app store reviews)
- Kakeibo methodology as a design framework

## Delegator Responsibilities
- Final approval on roadmap priorities
- Resolve conflicts between simplicity and user requests
- Provide business context for prioritization decisions

## Competencies, Qualities, and Skills
- Product thinking: ability to separate user needs from feature requests
- BDD scenario writing
- Understanding of Kakeibo methodology
- Empathy for users managing personal finances

## Key Metrics and Monitoring

| Metric | How Measured | Frequency | Measured By |
|--------|-------------|-----------|-------------|
| Backlog health | Stories have acceptance criteria and trace to requirements | Monthly | Product Lead |
| Feature adoption | % of users using new features within 30 days | Per release | Firebase Analytics |
| User satisfaction | Support ticket sentiment, app store ratings | Monthly | Product Lead |

## Evaluation Schedule
Quarterly review by Alex. Assess: Is the backlog serving the primary driver? Are shipped features being used? Is the product staying simple?
