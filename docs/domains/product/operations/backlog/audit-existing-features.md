# Backlog: Audit Existing Features

**Priority:** High
**Domain:** Product
**Requirement:** [Mindful Budget Planning](../../../governance/requirements/users-mindful-budgeting.md)

## Description
Audit the current MoneyDo app to document all existing features, their state, and how they map to the Kakeibo flow. This is the foundation for creating the user journey and story map.

## Acceptance Criteria
- [ ] Complete feature inventory with screenshots/descriptions
- [ ] Each feature mapped to a Kakeibo flow step
- [ ] Gaps identified (features missing from the ideal Kakeibo experience)
- [ ] User journey draft created from the audit
