# Policy: Decision-Making Process

**Date:** 2026-04-05
**Status:** Active

## Purpose
Define how decisions are made in MoneyDo — when agents can decide autonomously, when consent is needed, and how objections are resolved.

## Intended Outcome
Decisions happen at the right level: agents move fast on operational work within their domain, while governance and cross-domain decisions get proper review.

## Policy Details

### Operational decisions (within domain)
- Agent decides autonomously within their role description boundaries
- No approval needed for day-to-day work that follows existing specs
- Document non-obvious choices in PR descriptions

### Governance decisions (new policies, strategy changes, domain changes)
- Proposed by any agent as a governance backlog item
- Alex reviews using consent decision-making: "Is there a reason this would cause harm or miss an improvement?"
- If no objection → approved. If objection → resolve before proceeding.

### Cross-domain decisions
- Flagged by Coordinator or the affected agent
- Both domain owners consulted
- Alex makes the final call if agents can't align

### Objection resolution
1. Agent raises the objection with specific reasoning
2. If within domain → agent proposes an alternative
3. If cross-domain → Coordinator facilitates, Alex decides
4. Objections must be evidence-based — "I think" is a concern, not an objection

## Responsibilities

| Who | What |
|-----|------|
| Each agent | Raise objections when they see harm; decide within domain autonomy |
| Coordinator | Facilitate cross-domain decisions, flag unresolved conflicts |
| Alex (Founder) | Final authority on governance decisions and unresolved objections |

## Metrics and Monitoring

| Metric | How Monitored | Frequency |
|--------|--------------|-----------|
| Decision speed | Time from proposal to resolution | Quarterly |
| Objection quality | Are objections evidence-based and resolved constructively? | Quarterly |
